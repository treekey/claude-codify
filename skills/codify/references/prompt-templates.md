# E. 派工 prompt 模板

目標：產出專案的 `.claude/rules/prompt-templates.md`，內含五類常見任務的填空模板。派工時整份填完後送出，不得省略驗收與回報段。模板中 <> 為必填空格，[] 為視情況填。

## 通用骨架（五類模板都基於此）

```
任務類型：<搜尋|實作|重構|研究|審查>
目標：<一句話，完成後的狀態>
範圍：<允許讀的路徑> / <允許改的路徑（搜尋與研究類填「不得修改任何檔案」）>
已知背景：<派出方已確認的事實，弱模型不需重查>
驗收：<可機械檢查的條件清單>
回報格式：<固定結構>
升級條件：遇到 .claude/rules/rubrics.md Rubric 1 任一情況，停止並依交接格式回報。
```

## 1. 搜尋（找到位置，不改東西）

- 目標例：「找出所有讀取環境變數 FOO 的位置」
- 驗收：回報每個位置為 `檔案:行號 + 該行原文`；宣告搜尋完備的依據（用了哪些關鍵字與變形）。
- 回報格式：位置清單 + 搜尋詞清單 + 一句話結論。
- 注意：要求列出「搜過但沒中的變形」，避免弱模型搜一個詞就宣告完成。

## 2. 實作（新功能，範圍已定義）

- 前置：目標必須已由中高階模型收斂成無歧義描述，否則先退回。
- 驗收：<測試指令>全綠 + 新行為有對應測試 + diff 限於範圍段所列路徑。
- 回報格式：改動檔案清單（每檔一句原因）+ 測試輸出原文 + 未處理的邊界情況清單。

## 3. 重構（行為不變，結構改變）

- 驗收：重構前先跑 <測試指令> 記錄基線，重構後結果與基線完全一致；公開介面 signature 不變（或列出經核准的變更）。
- 回報格式：基線輸出 + 重構後輸出 + 結構變更摘要。
- 注意：模板內明寫「發現既有 bug 時記錄並回報，不要順手修」。

## 4. 研究（回答問題，產出結論）

- 驗收：每個結論附出處（檔案:行號 或 URL）；區分「證據支持」與「推測」兩節；至少列一個反面證據或明寫「未找到反例」。
- 回報格式：結論（≤5 條）→ 證據表 → 推測與不確定處。

## 5. 審查（對照 rubric 檢查產出）

- 範圍：只拿到「三要素 + 產出」，不拿執行者的推理過程。
- 驗收：對照 `.claude/rules/rubrics.md` Rubric 4 逐項檢查，每項給「通過／不通過＋證據」。
- 回報格式：三選一結論（通過／不通過附違反項／驗收條件本身有問題）+ 逐項檢查表。

## 填寫判準

- 「範圍」永遠明列路徑，不寫「相關檔案」。
- 「已知背景」寧多勿少：派出方多花 10 行背景，省下執行方整輪重新探索。
- 驗收條件寫完後自問：我收到回報時，能否不重做一遍就判斷真假？不能 → 驗收條件重寫。
